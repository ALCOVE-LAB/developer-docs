# 在 Localnet 上开发

## 推荐

大多数开发者应该使用 CLI 运行本地开发网络。
它比单一节点的 Localnet + Faucet 更简单且功能更全面。
如果你想要一个与生产网络工作方式相同的本地堆栈（local stack）
（[节点 API](../../build/apis.mdx) + [交易流](../../build/indexer/txn-stream.mdx) +
[索引器 API](../../build/indexer.mdx) + Faucet），这是你的选择。

- ### [运行本地开发网络](localnet/local-development-network.mdx)

## 特殊需求

如果你正在开发核心 Aptos 节点软件本身或有复杂的测试需求，请考虑以下指南。
