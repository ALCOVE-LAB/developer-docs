# 运行验证者和验证者全节点(VFN)

要参与 Aptos 共识协议，您必须部署并运行一个验证者节点和（可选）验证者全节点（VFN）。本文档提供了部署这两种节点类型所需的重要步骤的高级概述。

<Callout type="info">
虽然 VFN 不是参与共识所必需的，但强烈建议每个验证节点的运营者运行一个 VFN。

这是因为 VFN 是区块链数据进入生态系统的唯一入口和出口。网络中拥有许多 VFN 有助于提高网络可靠性、增加数据可用性，并为大众提供高质量的区块链访问体验。
</Callout>

在初始化质押池或委托池之前，请阅读有关 [质押](../blockchain/staking.mdx) 和 [委托质押](../blockchain/delegated-staking.mdx) 的信息，了解质押池类型之间的区别。请注意，一旦创建了质押池，就不能更改为委托池，反之亦然。另见 [外部资源](../../community/external-resources.md)，了解其他节点运营者提供的工具和服务。

使用本节中的文档来运行 Aptos 验证者和 VFN。在高级层面上，流程如下：

<Steps>
### 理解要求和部署类型

阅读节点要求并选择部署方法（例如，本地部署或云服务）。
首先阅读节点要求，了解所需的计算、内存、网络和存储资源。
此外，选择部署节点的方法，即使用云管理的 Kubernetes、Docker 或源代码。

### 为你的节点生成身份

为你的验证者和 VFN 创建公钥/私钥对和账户地址。
记得不要泄漏你的私钥！

### 配置你的验证者和 VFN

配置你的节点，以使用生成的密钥和身份。
使用 YAML 文件，根据前一步生成的密钥和身份配置你的节点。
这是为了让你的节点能够安全地连接到其他节点（即，对等节点，peers）。

### 为你的节点下载创世文件和路标（genesis and a waypoint）

使用创世文件和路标引导节点，并为启动做准备。
节点配置正确后，安装二进制文件并下载创世块和路标文件。
这些将为你的节点提供它们需要的信息，以便开始与其他节点同步。

### 加入验证者集合并开始参与

初始化质押池，加入验证者集合并开始验证以获得奖励。
在你的节点被其他节点接受连接之前，你需要加入验证者集合。
为此，你必须初始化一个质押池，并向运营商和投票者委托。
一旦你的质押池设置完成，你就可以加入验证者集合。
此时，你的节点将开始与网络同步，你的验证者将能够开始参与共识。
这时，你可以开始获得奖励。

</Steps>
